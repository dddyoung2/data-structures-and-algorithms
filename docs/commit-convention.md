# Commit Convention

## English Version

### Type
- docs: add or update concept notes and README files
- feat: add a new implementation or problem solution
- fix: fix code errors
- test: add test code
- refactor: improve code structure
- chore: organize folders or update configuration

### Examples
```bash
git commit -m "docs: add graph concept note"
git commit -m "feat: implement BFS traversal"
git commit -m "feat: solve LeetCode 1971 with BFS"
git commit -m "fix: prevent duplicate queue insertion"
```

---

### 종류
- docs: 개념 정리, README 수정
- feat: 새 구현 또는 새 문제풀이 추가
- fix: 코드 오류 수정
- test: 테스트 코드 추가
- refactor: 코드 구조 개선
- chore: 폴더 정리, 설정 변경

### 예시
```bash
git commit -m "docs: add graph concept note"
git commit -m "feat: implement BFS traversal"
git commit -m "feat: solve LeetCode 1971 with BFS"
git commit -m "fix: prevent duplicate queue insertion"
```
